# Кэш

При `cache: true` apig кэширует распарсенный IR (промежуточное представление), чтобы не скачивать и не парсить спецификацию заново при каждом запуске, если она не изменилась.

## Конфигурация

```ts
defineConfig({
  input: 'https://api.example.com/openapi.json',
  output: './src/api',
  cache: true,
  plugins: [typescript(), sdk()],
})
```

## Как это работает

**Удалённые спецификации (URL):**

1. При первом запуске — скачивает и парсит спецификацию, сохраняет IR и ETag в `.apig/cache/`
2. При последующих — отправляет заголовок `If-None-Match` с кэшированным ETag
3. Если сервер возвращает `304 Not Modified` — использует закэшированный IR, пропускает повторное скачивание и парсинг
4. Если ETag изменился — скачивает и парсит обновлённую спецификацию, обновляет кэш

**Локальные файлы:**

1. При первом запуске — вычисляет SHA-256 хэш содержимого файла, сохраняет вместе с IR
2. При последующих — сравнивает текущий хэш с кэшированным
3. Если не изменился — использует закэшированный IR, пропускает парсинг
4. Если изменился — парсит заново и обновляет кэш

Генерация файлов из IR (шаг записи на диск) выполняется в любом случае — `cache` экономит только сетевой запрос и парсинг спецификации.

## Расположение кэша

```
.apig/
  cache/
    <key>.ir.json     — закэшированный IR
    <key>.meta.json   — ETag (для URL) или хэш спецификации (для локальных файлов)
```

`<key>` — первые 16 символов SHA-256 хэша от значения `input`.

Добавь в `.gitignore` если не хочешь коммитить кэш:

```
.apig/cache/
```
