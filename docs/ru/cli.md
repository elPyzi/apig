# CLI

## Команды

### `apig generate` (алиас `g`)

Запуск генерации кода.

```bash
apig generate
```

**Флаги:**

| Флаг | Описание |
|------|----------|
| `-c, --config <путь>` | Путь к файлу конфигурации (по умолчанию `apig.config.ts`) |
| `-d, --dry-run` | Предпросмотр без записи файлов |
| `-w, --watch` | Отслеживание изменений спецификации и конфига |

```bash
apig generate --dry-run
apig generate --watch
apig generate -c ./config/apig.config.ts
```

При `--watch` для локальных файлов спецификации отслеживаются изменения и конфига, и самого файла спецификации. Для спецификаций по URL отслеживание файла не поддерживается — только конфига.

---

### `apig start` (алиас `s`)

Интерактивный мастер настройки — выбор плагинов и параметров вывода через диалоговые вопросы в терминале.

```bash
apig start
```

---

### `apig config` (алиас `c`)

Создаёт файл `apig.config.ts` в текущей директории. Команда **не** выводит текущий конфиг — она генерирует новый файл-заготовку.

```bash
apig config
```

Если `apig.config.ts` уже существует — команда завершится с ошибкой.

**Флаги:**

| Флаг | Описание |
|------|----------|
| `-p, --preset <name>` | Использовать один из готовых пресетов |
| `--list-presets` | Показать список доступных пресетов и выйти |

```bash
apig config --list-presets
apig config --preset react
```

**Доступные пресеты:**

| Пресет | Описание |
|--------|----------|
| `minimal` | TypeScript типы + SDK fetch-функции |
| `react` | TypeScript + SDK + TanStack Query + Zod (стандартный React стек) |
| `react-swr` | TypeScript + SDK + SWR + Zod |
| `testing` | React стек + Faker фабрики + MSW обработчики для мокинга |
| `forms` | TypeScript + SDK + Zod + резолверы React Hook Form |
| `full` | Все плагины сразу |

---

### `apig info` (алиас `i`)

Показывает статистику по загруженной спецификации без генерации файлов: название и версию API, путь ввода/вывода, `groupBy`, количество операций (с разбивкой по методам), количество схем, список тегов, подключённые плагины.

```bash
apig info
```

**Флаги:**

| Флаг | Описание |
|------|----------|
| `-c, --config <путь>` | Путь к файлу конфигурации (по умолчанию `apig.config.ts`) |

---

### `apig versions`

Список всех сохранённых снапшотов (требует включённого `versioning.enabled` в конфиге).

```bash
apig versions
```

**Флаги:**

| Флаг | Описание |
|------|----------|
| `-c, --config <путь>` | Путь к файлу конфигурации |
| `-s, --storage <путь>` | Директория хранилища снапшотов (переопределяет конфиг) |

Колонки: алиас, ID снапшота, дата создания.

---

### `apig version checkout <id|alias>`

Перегенерация кода из сохранённого снапшота.

```bash
apig version checkout abc123
apig version checkout gen5
```

**Флаги:** `-c, --config <путь>`, `--dry-run`

---

### `apig version show <id|alias>`

Подробная информация о снапшоте: алиас, ID, версия API, номер генерации, дата создания, комментарий, наличие сохранённой спецификации.

```bash
apig version show abc123
```

**Флаги:** `-c, --config <путь>`, `-s, --storage <путь>`

---

## Кастомный путь к конфигу

По умолчанию apig ищет `apig.config.ts` в текущей директории. Используй `-c` для указания другого файла:

```bash
apig generate -c ./config/apig.config.ts
```
