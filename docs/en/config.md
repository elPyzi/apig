# Configuration

`apig.config.ts` is the entry point for all configuration. Export a single config or an array of configs.

```ts
import { defineConfig } from '@travjek/apig'

export default defineConfig({
  input: './openapi.json',
  output: './src/api',
  plugins: [],
})
```

Multiple configs (e.g. for several APIs in one project):

```ts
export default [
  defineConfig({ input: './users-api.json', output: './src/users' }),
  defineConfig({ input: './orders-api.json', output: './src/orders' }),
]
```

## Options

### `name`

**Type:** `string`

A label for this config, shown in CLI output when multiple configs are used.

```ts
name: 'users-api'
```

---

### `input`

**Type:** `string | (() => Promise<string>)`  
**Required:** yes

A path, URL, or async function returning a path/URL to the OpenAPI spec. Supports OpenAPI 3.0 and Swagger 2.0 (auto-upgraded).

```ts
input: './openapi.json'
input: 'https://api.example.com/openapi.json'
input: () => fetch('https://api.example.com/openapi.json').then(r => r.json())
```

---

### `output`

**Type:** `string | { path: string; clean?: boolean }`  
**Default:** `.apig/generated`

Directory where generated files are written. When a string is passed, `clean` defaults to `true` (the directory is cleared before generation).

```ts
output: './src/api'
output: { path: './src/api', clean: false }
```

---

### `httpClient`

**Type:** `{ name: 'fetch' | 'axios' | 'ky' | 'ofetch' | 'wretch'; path?: string; export?: string }`  
**Default:** `{ name: 'fetch' }`

The HTTP client used by functions generated by `sdk()`. For `axios`, `ky`, `ofetch`, `wretch`, `path` and `export` are required — the file path and named export of your client instance. Not needed for `fetch`.

```ts
httpClient: { name: 'axios', path: './lib/axios', export: 'axiosInstance' }
```

---

### `plugins`

**Type:** `ApigPlugin[]`

The plugin array. See [Plugins](./plugins/index.md) for the full list.

---

### `baseUrl`

**Type:** `string`

Prefix prepended to all generated request paths.

```ts
baseUrl: '/api/v1'
baseUrl: 'https://api.example.com'
```

---

### `groupBy`

**Type:** `'none' | 'tags' | 'endpoints' | 'operations'`  
**Default:** `'none'`

Controls how generated files are split:

- `none` — everything in `output`, no subfolders (`sdk.ts`, `types.ts`, ...)
- `tags` — one subfolder per OpenAPI tag (`users/users.sdk.ts`)
- `endpoints` — one subfolder per tag, with a subfolder per operation inside it (`users/get-user/get-user.sdk.ts`)
- `operations` — one subfolder per operation, no tag grouping (`get-user/get-user.sdk.ts`)

---

### `fileNaming`

**Type:** `'camelCase' | 'kebab-case' | 'snake_case' | 'PascalCase'`  
**Default:** `'kebab-case'`

Naming convention for generated files and directories (used when `groupBy` is not `none`).

---

### `functionNaming`

**Type:** `'camelCase' | 'kebab-case' | 'snake_case' | 'PascalCase'`  
**Default:** `'camelCase'`

Naming convention for generated SDK function names.

---

### `enumStyle`

**Type:** `'union' | 'enum' | 'const'`

Controls enum generation across `typescript`, `zod`, `valibot`, and `yup` plugins. This is a global `defineConfig` option, not a per-plugin option.

- `union` — `type Status = 'active' | 'inactive'`
- `enum` — `enum Status { Active = 'active' }`
- `const` — `const Status = { Active: 'active' } as const`

By default, `typescript()` uses `'const'`, while `zod()`/`valibot()`/`yup()` use `'union'` when `enumStyle` isn't set explicitly.

---

### `typeStyle`

**Type:** `'type' | 'interface'`  
**Default:** `'type'`

Type declaration style for object schemas in `typescript()`. This is a global `defineConfig` option.

- `type` — `export type User = { ... }`
- `interface` — `export interface User { ... }`

---

### `filter`

**Type:** `{ tags?: string[], exclude?: string[], deprecated?: boolean }`

Include or exclude operations:

```ts
filter: {
  tags: ['users', 'orders'], // only generate these tags
  exclude: ['internal'],     // exclude these tags
  deprecated: false,         // include deprecated operations (default false)
}
```

---

### `rename`

**Type:** `Record<string, string>`

Rename `operationId` values before generation:

```ts
rename: {
  get_users_list: 'getUsers',
  create_user_v2: 'createUser',
}
```

---

### `validate`

**Type:** `boolean`

> **Known quirk:** this field is currently not read anywhere in code generation besides a type-check during config validation — it has no effect on actual behavior. Don't rely on it.

---

### `endpointsMap`

**Type:** `boolean`  
**Default:** `false`

Generate an `endpoints.ts` file with a typed constant of all API paths. See [endpointsMap](./endpoints-map.md).

---

### `index`

**Type:** `boolean`  
**Default:** `true`

Controls generation of `index.ts` with re-exports of all generated files.

---

### `formatter`

**Type:** `'prettier' | 'biome' | 'oxfmt' | 'none'`  
**Default:** `'none'`

Formats generated files after writing.

---

### `cache`

**Type:** `boolean`  
**Default:** `false`

Cache the parsed IR on disk (`.apig/cache`). On repeated runs, skips parsing if the spec hasn't changed (ETag for URLs, hash for local files). See [Cache](./cache.md).

---

### `apiLogging`

**Type:** `boolean`  
**Default:** `false`

Adds `console.log(functionName, response)` to every generated SDK function.

---

### `cliLogging`

**Type:** `{ level?: 'minimal' | 'normal' | 'detailed' }`  
**Default:** `{ level: 'minimal' }`

CLI logging verbosity during generation.

---

### `errorHandling`

**Type:** `boolean | { path: string; export: string }`  
**Default:** `true`

Error handling for SDK functions:

- `true` (default) — generates a built-in `ApigError` class into `config.ts` in the output directory
- `false` — disables error handling entirely
- `{ path, export }` — uses your custom error class from the given path

```ts
errorHandling: { path: './lib/errors', export: 'ApiError' }
```

See [ApigError client](./client.md).

---

### `rawResponse`

**Type:** `boolean`  
**Default:** `false`

Return the full response object `{ body, status, headers }` instead of just the data.

---

### `hooks`

**Type:** `{ afterAllFilesWrite?: string }`

Shell command run after all files are written:

```ts
hooks: {
  afterAllFilesWrite: 'eslint ./src/api --fix',
}
```

---

### `comment`

**Type:** `string`

Comment attached to the versioning snapshot on the next generation run.

```ts
comment: 'before auth refactor'
```

---

## Versioning

**Type:** `Versioning`

```ts
versioning: {
  enabled: true,
  storage: '.apig/versions',
  maxSaves: 10,
  saveSpec: true,
  pinVersions: ['abc123'],
  aliasTemplate: 'v{apiVersion}-gen{generation}',
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `enabled` | `boolean` | `false` | Enable snapshot versioning |
| `storage` | `string` | `.apig/versions` | Directory to store snapshots |
| `saveSpec` | `boolean` | `false` | Save the original OpenAPI spec in each snapshot |
| `maxSaves` | `number` | unlimited | Maximum snapshots to keep; oldest removed first |
| `pinVersions` | `string[]` | — | Snapshot IDs that are never automatically deleted |
| `aliasTemplate` | `string` | `gen{generation}` | Snapshot alias template |

Available template variables: `{generation}`, `{apiVersion}`, `{date}`.
