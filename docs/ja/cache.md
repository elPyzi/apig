# キャッシュ

`cache: true` を指定すると、apig はパース済みのIR（中間表現）をキャッシュし、仕様が変更されていない限り、実行するたびに再ダウンロードや再パースを行わないようにする。

## 設定

```ts
defineConfig({
  input: 'https://api.example.com/openapi.json',
  output: './src/api',
  cache: true,
  plugins: [typescript(), sdk()],
})
```

## 仕組み

**リモート仕様（URL）:**

1. 初回実行時 — 仕様をダウンロード・パースし、IRとETagを`.apig/cache/`に保存
2. 2回目以降 — キャッシュ済みのETagを`If-None-Match`ヘッダーで送信
3. サーバーが`304 Not Modified`を返した場合 — キャッシュ済みIRを使用し、再ダウンロードと再パースをスキップ
4. ETagが変わっていた場合 — 更新された仕様をダウンロード・パースし、キャッシュを更新

**ローカルファイル:**

1. 初回実行時 — ファイル内容のSHA-256ハッシュを計算し、IRと一緒に保存
2. 2回目以降 — 現在のハッシュとキャッシュ済みハッシュを比較
3. 変更がない場合 — キャッシュ済みIRを使用し、パースをスキップ
4. 変更があった場合 — 再パースしてキャッシュを更新

IRからのファイル生成（ディスクへの書き込みステップ）はいずれの場合も実行される — `cache`が節約するのはネットワークリクエストと仕様のパースのみ。

## キャッシュの保存場所

```
.apig/
  cache/
    <key>.ir.json     — キャッシュされたIR
    <key>.meta.json   — ETag（URLの場合）または仕様のハッシュ（ローカルファイルの場合）
```

`<key>` は `input` の値のSHA-256ハッシュの先頭16文字。

キャッシュをコミットしたくない場合は`.gitignore`に追加する。

```
.apig/cache/
```
